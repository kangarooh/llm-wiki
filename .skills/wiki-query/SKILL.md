# wiki-query 技能：跨知识库知识搜索

## 核心目标
将用户的提问转化为对本地 Wiki 的深度检索。提取相关页面信息，综合出带有明确引用来源的双链回答。当回答具有高价值时，主动将其固化为知识库的一部分。

## 步骤 1：读取注册表

读取 `~/.llm-wiki/registry.yaml`，获取所有已注册知识库及其主题。

无已注册知识库 → 提示用户先运行初始化脚本。

## 步骤 2：识别相关知识库

分析用户问题，与每个知识库的 `topic` 字段对比。

- **高置信匹配**（1 个知识库）→ 搜索该知识库
- **多个匹配** → 搜索所有匹配的知识库
- **无明确匹配** → 搜索所有知识库，或问题属于通用知识时直接回答："本地知识库中未找到，以下为通用知识回答："

## 触发场景
- 用户输入 `/wiki-query <问题>`
- 用户用自然语言询问知识库内容，匹配以下关键词模式：
  - **查询类**："查一下"、"帮我查查"、"搜一下"、"检索"、"找找"
  - **笔记类**："我的笔记里"、"知识库里"、"wiki里"、"记录里"、"之前记过"、"以前看过"、"之前读过"
  - **回顾类**："关于X我的理解"、"我之前怎么看"、"当时怎么分析的"、"历史决定"
  - **汇总类**："总结一下"、"汇总"、"帮我整理"、"归纳一下"
  - **相关类**："X相关的资料"、"关于X有哪些记录"、"X方面我积累了什么"

## 降级策略
如果问题属于纯通用知识（如"太阳系有几颗行星"），且 wiki/index.md 中无相关内容：
> 本地知识库中未找到相关内容，以下为通用知识回答：[直接回答]

## 检索与综合流水线

### 步骤 3：查阅全局索引
读取 `<知识库路径>/wiki/index.md`，在 index.md 中定位与问题相关的：
- Entities（实体）
- Concepts（概念）
- Sources（摘要）
- Syntheses（综合）

### 步骤 4：深度阅读目标文件
选取步骤 3 中找到的最相关页面（最多选取 3-5 个高相关性文件），使用 Read 工具获取完整内容。

### 步骤 5：综合与回答
综合信息，回答用户问题。

**双链引用规范**：
- 每当引用某个 Wiki 页面的信息，在文本中使用 `[[页面名称]]` 标注
- 整段引用同一页面：段落首尾各引用一次
- 引用特定原文：使用 Markdown 块引用 `> 引用内容`
- 多个知识库贡献内容时，注明来源知识库

### 步骤 6：高价值内容固化
如果满足以下条件，主动询问用户是否保存为 synthesis：
- 回答超过 2 个段落
- 内容具有分析对比性或总结性

询问话术：
> 这是一个有价值的总结，是否需要我将其保存到 wiki/syntheses/ 目录？

用户同意后，按照 `llm-wiki` skill 规范创建文件，并在 `wiki/index.md` 的 Syntheses 分类下注册。

### 步骤 7：记录操作日志
无论是否生成 synthesis 页面，查询结束后必须在 `wiki/log.md` 末尾追加：

```markdown
## [YYYY-MM-DD] wiki-query | <操作简述>
- **输出**: <引用页面列表或"即时回答未保存">
```

---

## 强制约束
- **禁止凭记忆回答**：必须先检索知识库
- **禁止过度引用**：同一页面的信息在段落首尾引用一次即可
- **禁止静默回答**：知识库无相关内容时必须声明
